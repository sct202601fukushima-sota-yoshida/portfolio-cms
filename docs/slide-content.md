## 登録手順

1. `/admin/login` にログイン（`admin` / `admin123`）
2. まず**カテゴリ**を6件作成（下記「カテゴリ定義」の通り）
3. 各カテゴリの下に、本ドキュメントの順に**スライド**を作成
4. `sortOrder` は各カテゴリ内の並び順（0始まり）
5. `isActive` はすべて `true`

# カテゴリ5: ポートフォリオ

## スライド 5-0

- **categoryName**: ポートフォリオ
- **sortOrder**: 0
- **title**: このサイト自体が成果物です
- **description**:

今ご覧いただいている本サイトは、**静的なHTMLではなく**、Spring Boot 製のフルスタックアプリケーションです。
このページに並ぶスライド一枚一枚は、管理画面（`/admin`）から CRUD 操作で登録・編集・削除されたデータベース上のレコードです。
「自己紹介サイト」と「CMS」と「技術デモ」を**1つのプロダクトに統合した**構成で、
デザイン科出身ならではの**見た目**と、職業訓練で鍛えた**設計・実装**の両面を、同時にご覧いただけます。

## スライド 5-1

- **categoryName**: ポートフォリオ
- **sortOrder**: 1
- **title**: 技術スタックと構成
- **description**:

本サイトは以下の構成で実装しております。

- **4テーブル正規化**：User / Category / Slide / SlideImage の 1対多リレーション
- **レイヤ分離**：Controller → Service → Repository → Entity
- **認証**：Spring Security + BCrypt、管理画面は ROLE_ADMIN 限定
- **論理削除**：`deleted_at IS NULL` 部分インデックスによる高速公開クエリ
- **テスト**：Service 層は Mockito、Controller 層は `@WebMvcTest` スライス
- **CI**：push 時に PostgreSQL サービスコンテナ上で Flyway + 全テスト実行

「動くCRUD」で終わらせず、**実運用で必要になる要素**を網羅しております。

## スライド 5-2

- **categoryName**: ポートフォリオ
- **sortOrder**: 2
- **title**: 工夫した点
- **description**:

本サイトで特に注力したポイントです。

- **論理削除 + 部分インデックス**：公開クエリに削除済みレコードを読ませない設計
- **テストスライスの最小化**：ファイルストレージ依存を除外し、Controller 層テストを DB なし・数百ms で実行
- **環境変数によるDB資格情報注入**：ローカル／CI／本番で同じ `application.yml` を使い回す構成
- **独自CSS**：Bootstrap に依存せず、`:root` の CSS 変数でデザインシステムを一元化

一人で通す過程で、**運用を見据えた判断**を繰り返してきました。
